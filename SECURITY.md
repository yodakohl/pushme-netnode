# Security Policy

## Supported versions

Security fixes are applied to:

- the current `main` branch
- the current published container release line, currently `0.3.x`

Older release lines may receive guidance, but they should not be assumed to get
patches.

## Reporting a vulnerability

Do not open a public issue for vulnerabilities.

Report vulnerabilities privately to `hello@pushme.site`. Include:

- the affected version or image tag
- reproduction steps or a proof of concept
- impact details
- any suggested mitigation if you already have one

If GitHub private vulnerability reporting is enabled for this repo, that path is
also acceptable.

## What to expect

- an acknowledgment within 3 business days
- a coordinated fix and release when the report is valid
- public disclosure after a fix is available or a mitigation is in place
